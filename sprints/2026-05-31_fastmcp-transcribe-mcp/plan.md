# Sprint: Create a FastMCP server (mcp_code/) exposing a transcribe tool that wraps the existing FastAPI /transcribe endpoint

**Started:** 2026-05-31
**Goal:** Build a FastMCP server under `mcp_code/` that exposes a `transcribe` tool which takes a local audio file path (English or Hebrew request phrasing), POSTs the file to the configurable FastAPI `/transcribe` base URL, and returns the JSON result.

## Context (read before starting)

- **Existing API contract** (built last sprint, see `app/main.py`): `POST /transcribe` accepts `multipart/form-data` with file field name **`file`**, returns JSON:
  `{ "segments": [{"speaker": str, "text": str, "start": float, "end": float}, ...], "language": str|null, "num_speakers": int|null }`.
  There is also `GET /health` -> `{"status": "ok"}`. Default base URL `http://localhost:8000` (FastAPI HOST default `0.0.0.0`, PORT `8000`).
- **Reference HTTP call**: `client_example.py` (repo root) already implements the exact request shape the MCP tool must reproduce — see its `transcribe()` (lines 81-91): `files = {"file": (audio_path.name, fileobj, "audio/m4a")}`, `requests.post(f"{base_url}/transcribe", files=files, timeout=...)`. Reuse this pattern; do NOT re-derive the contract.
- **Build skill**: The implementer MUST invoke the **`mcp-rest-builder`** skill ("build a FastMCP server that wraps an existing REST API"). That skill's build/CLI workflow is conducted in **English** (the CLI does not render Hebrew well). This English-build constraint applies to the *build process only*. The **end user** of the finished tool must be able to phrase a transcription request in **English OR Hebrew**, so the tool's docstring/description must include both-language cues (see T2) so the model routes Hebrew requests to this tool.
- **Tech**: repo uses `uv`, Python >=3.11, `pyproject.toml`. `requests` is already a dependency. `fastmcp` must be added (T1). Tests use `pytest` (already in dev group) + `httpx` (already in dev group); HTTP calls in tests are **mocked** (no live server, no real model load).
- **Folder is new**: `mcp_code/` does not exist yet. Keep it self-contained: server module, `__init__.py`, tests under `mcp_code/tests/` (or `tests/` — match the existing repo convention, which is a top-level `tests/` package; the implementer/architect decides and documents in T2).
- **Config**: API base URL must be read from an env var (e.g. `TRANSCRIBE_API_URL`) with default `http://localhost:8000`. The server must import and start with zero env vars set.
- **Sample audio** for any manual smoke test: `resource/audio smaples/audio_sample_1min.m4a` (note the space and "smaples" typo — quote the path).
